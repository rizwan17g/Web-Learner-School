What are CSS Selectors?

CSS selectors are patterns used to select and target HTML elements that you want to style. Think of selectors as a way to "point" to specific elements on your page and say "apply these styles to these elements."

Selectors are the first part of every CSS rule and determine which elements will receive the styling. Mastering selectors gives you precise control over your page's appearance and allows you to create sophisticated, well-organized stylesheets.

There are many different types of selectors, from simple element selectors to complex combinations that can target elements based on their position, attributes, or relationship to other elements.

Key Concept: Selectors are the bridge between your CSS styles and your HTML elements. The more precisely you can select elements, the more control you have over your design.

Basic Selectors

Let's start with the fundamental selectors that form the foundation of CSS styling:

Element Selector

Selects all elements of a specific HTML tag type. This is the most basic selector.

Element Selector Examples
/* Select all paragraphs */
p {
    color: blue;
    margin-bottom: 15px;
}

/* Select all headings level 1 */
h1 {
    font-size: 2rem;
    text-align: center;
}

/* Select all links */
a {
    color: #3498db;
    text-decoration: none;
}

/* Select all images */
img {
    max-width: 100%;
    height: auto;
}

Class Selector

Selects elements with a specific class attribute. Classes are reusable and can be applied to multiple elements.

Class Selector Examples
/* Select elements with class="highlight" */
.highlight {
    background-color: yellow;
    padding: 10px;
}

/* Select elements with class="button" */
.button {
    display: inline-block;
    padding: 12px 24px;
    background-color: #007bff;
    color: white;
    border-radius: 4px;
}

/* Select elements with class="warning" */
.warning {
    border: 2px solid red;
    background-color: #ffe6e6;
    padding: 15px;
}

ID Selector

Selects a single element with a specific ID attribute. IDs should be unique on a page.

ID Selector Examples
/* Select element with id="header" */
#header {
    background-color: #2c3e50;
    color: white;
    padding: 20px;
}

/* Select element with id="main-content" */
#main-content {
    max-width: 800px;
    margin: 0 auto;
    padding: 40px 20px;
}

/* Select element with id="footer" */
#footer {
    background-color: #34495e;
    text-align: center;
    padding: 30px;
}

Combinator Selectors

Combinator selectors target elements based on their relationship to other elements in the HTML structure:

Descendant Selector (Space)

Selects elements that are descendants of another element, regardless of how deeply nested they are.

Descendant Selector Examples
/* Select all paragraphs inside articles */
article p {
    font-size: 16px;
    line-height: 1.6;
}

/* Select all links inside navigation */
nav a {
    color: white;
    text-decoration: none;
    padding: 10px 15px;
}

/* Select all images inside figure elements */
figure img {
    width: 100%;
    border-radius: 8px;
}

Child Selector (>)

Selects elements that are direct children of another element.

Child Selector Examples
/* Select only direct paragraph children of main */
main > p {
    margin-bottom: 20px;
}

/* Select only direct list items of nav ul */
nav ul > li {
    display: inline-block;
    margin-right: 20px;
}

/* Select only direct div children of container */
.container > div {
    flex: 1;
    margin: 10px;
}

Adjacent Sibling Selector (+)

Selects an element that immediately follows another element.

Adjacent Sibling Selector Examples
/* Select paragraphs immediately after h2 */
h2 + p {
    font-weight: bold;
    margin-top: 0;
}

/* Select images immediately after paragraphs */
p + img {
    margin-top: 20px;
}

/* Select divs immediately after headers */
header + div {
    padding-top: 30px;
}

General Sibling Selector (~)

Selects all elements that are siblings of another element.

General Sibling Selector Examples
/* Select all paragraphs that are siblings after h2 */
h2 ~ p {
    color: #666;
}

/* Select all images that are siblings after a paragraph */
p ~ img {
    border: 1px solid #ddd;
}

/* Select all sections that are siblings after header */
header ~ section {
    margin-top: 40px;
}

Attribute Selectors

Attribute selectors target elements based on their attributes and attribute values:

Attribute Selector Examples
/* Select elements with a specific attribute */
[title] {
    border-bottom: 1px dotted;
}

/* Select elements with a specific attribute value */
[type="email"] {
    border: 2px solid blue;
}

/* Select elements where attribute begins with a value */
[href^="https"] {
    color: green;
}

/* Select elements where attribute ends with a value */
[src$=".jpg"] {
    border: 2px solid red;
}

/* Select elements where attribute contains a value */
[class*="btn"] {
    padding: 10px 20px;
    border-radius: 4px;
}

/* Select elements where attribute contains a word */
[class~="active"] {
    background-color: #007bff;
    color: white;
}

Pseudo-Classes

Pseudo-classes select elements based on their state or position, not their attributes or content:

Link and User Action Pseudo-classes

Link and Action Pseudo-classes
/* Unvisited links */
a:link {
    color: #007bff;
}

/* Visited links */
a:visited {
    color: #6f42c1;
}

/* Hover state */
a:hover {
    color: #0056b3;
    text-decoration: underline;
}

/* Active state (being clicked) */
a:active {
    color: #004085;
}

/* Focus state (keyboard navigation) */
a:focus {
    outline: 2px solid #007bff;
    outline-offset: 2px;
}

Structural Pseudo-classes

Structural Pseudo-classes
/* First child element */
li:first-child {
    font-weight: bold;
}

/* Last child element */
li:last-child {
    border-bottom: none;
}

/* Nth child elements */
tr:nth-child(even) {
    background-color: #f8f9fa;
}

tr:nth-child(odd) {
    background-color: white;
}

/* First of type */
p:first-of-type {
    font-size: 1.2em;
}

/* Only child */
div:only-child {
    text-align: center;
}

Form Pseudo-classes

Form Pseudo-classes
/* Required fields */
input:required {
    border-left: 3px solid red;
}

/* Valid fields */
input:valid {
    border-left: 3px solid green;
}

/* Invalid fields */
input:invalid {
    border-left: 3px solid red;
}

/* Disabled elements */
input:disabled {
    background-color: #e9ecef;
    cursor: not-allowed;
}

/* Checked checkboxes and radio buttons */
input:checked {
    transform: scale(1.1);
}

Pseudo-Elements

Pseudo-elements create virtual elements that you can style, allowing you to target specific parts of elements:

Pseudo-element Examples
/* Style the first line of paragraphs */
p::first-line {
    font-weight: bold;
    font-size: 1.1em;
}

/* Style the first letter of paragraphs */
p::first-letter {
    font-size: 2em;
    float: left;
    margin-right: 5px;
}

/* Insert content before elements */
.quote::before {
    content: '"';
    font-size: 2em;
    color: #666;
}

.quote::after {
    content: '"';
    font-size: 2em;
    color: #666;
}

/* Style placeholder text */
input::placeholder {
    color: #999;
    font-style: italic;
}

/* Create decorative elements */
.section-title::after {
    content: '';
    display: block;
    width: 50px;
    height: 3px;
    background-color: #007bff;
    margin-top: 10px;
}

Selector Specificity and Best Practices

Understanding how selectors work together and their relative specificity is crucial for maintainable CSS:

Specificity Calculation

  • Inline styles: 1000 points
  • IDs: 100 points each
  • Classes, attributes, pseudo-classes: 10 points each
  • Elements and pseudo-elements: 1 point each
Specificity Examples
/* Specificity: 1 (1 element) */
p { color: black; }

/* Specificity: 10 (1 class) */
.highlight { color: blue; }

/* Specificity: 11 (1 class + 1 element) */
p.highlight { color: green; }

/* Specificity: 100 (1 ID) */
#special { color: red; }

/* Specificity: 111 (1 ID + 1 class + 1 element) */
#special.highlight p { color: purple; }

/* Specificity: 21 (2 classes + 1 element) */
.container.main p { color: orange; }

Best Practices for Selectors

  • Keep specificity low: Use classes instead of IDs when possible
  • Avoid overly specific selectors: Don't use more specificity than needed
  • Use meaningful class names: Choose names that describe purpose, not appearance
  • Avoid !important: Solve specificity conflicts with better selectors
  • Group related selectors: Organize your CSS logically
Golden Rule: Write selectors that are as specific as needed, but no more specific than necessary. This keeps your CSS flexible and maintainable.

Advanced Selector Combinations

Combining different selector types gives you powerful targeting capabilities:

Advanced Selector Combinations
/* Multiple classes */
.button.primary {
    background-color: #007bff;
    color: white;
}

.button.secondary {
    background-color: #6c757d;
    color: white;
}

/* Class with pseudo-class */
.button:hover {
    transform: translateY(-2px);
    box-shadow: 0 4px 8px rgba(0,0,0,0.1);
}

/* Attribute selector with pseudo-class */
input[type="email"]:focus {
    border-color: #007bff;
    box-shadow: 0 0 0 2px rgba(0,123,255,0.25);
}

/* Complex combinations */
.article-content > p:first-child::first-letter {
    font-size: 3em;
    color: #007bff;
    float: left;
    margin-right: 8px;
}

/* Multiple selector grouping */
h1, h2, h3, h4, h5, h6 {
    font-family: 'Georgia', serif;
    font-weight: bold;
    margin-bottom: 0.5em;
}

/* Targeting specific form elements */
form input[type="text"]:required:invalid {
    border-color: #dc3545;
    background-color: #f8d7da;
}
Pro Tip: When combining selectors, read them from right to left. The browser processes selectors this way, and it helps you understand what elements will be selected.
